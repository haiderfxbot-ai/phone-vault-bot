Dashboard User Manual

This dashboard tracks every Termux command required to deploy the JARVIS Mythos Core from scratch on any Android device. Each card represents one atomic setup step. Use the guide below to understand the flow.

Environment Initialization (Step 1) Updates all Termux base packages to their latest versions. Run this first to avoid dependency conflicts.
System Dependency Installation (Step 2) Installs Python (interpreter), Git (version control), and curl (HTTP client) into the Termux environment.
Python Library Layer (Step 3) Pulls the three required modules: requests (HTTP client for Ollama API), colorama (terminal color output), and python-dotenv (environment variable loader).
Clone Repository (Step 4) Pulls the full Mythos Core codebase from GitHub into /storage/emulated/0/jarvis_ai_core. This overwrites any previous version at that path. After cloning, the shell is positioned inside the project directory.
Ollama Engine Deployment (Step 5) Installs Ollama via the Termux native package manager (pkg install ollama -y). This is the correct installation method for non-rooted Android devices — the standard curl bootstrap script requires superuser permissions and will fail.
Background Service Activation (Step 6) Launches the Ollama REST API server on 127.0.0.1:11434 as a background process. The ampersand (&) keeps it running after the terminal session continues.
Model Acquisition (Step 7) Pulls the llama3.2:3b model weights (3B params, ~2 GB RAM) into Ollama's local storage. This fits devices with ~3 GB free RAM. Download may take several minutes. The model is cached after download.
Core Execution (Step 8) Starts the custom Python REPL client. It connects to the local Ollama server, injects the uncensored Mythos system prompt, and opens an interactive session. Type exit to quit.
Offline Dry-Run Test (Step 9) Launches the same interface but in simulation mode. No Ollama connection is required — ideal for verifying the UI, color rendering, and command flow before the model is fully ready.

Persistent Tracking: Each card's green accent state is saved to your browser's localStorage. Close, refresh, or reopen the page — completed steps remain marked. Tap Copy on any command to copy it to your clipboard and mark it complete.
